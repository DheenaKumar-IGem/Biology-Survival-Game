# PDAC Immune Defense

An educational survival game about pancreatic ductal adenocarcinoma (PDAC)
and saliva-based early detection research. Players control an immune defender,
swap between immune-response weapons, fight themed biological threats, and
advance through short science lessons between combat rounds.

## Current Focus

The game is being pushed from a playable prototype toward a more polished
student-facing release:

- smoother web performance under large enemy waves
- clearer boss danger patterns
- more weapon/enemy variety across the campaign
- a professional startup/loading flow
- a stronger first impression from the home screen
- short mission briefings when each combat round begins
- a continue-run checkpoint flow for interrupted sessions

## Run The Game

```powershell
flutter pub get
flutter run -d chrome
```

For a production-style web build:

```powershell
flutter build web
python -m http.server 5517 -d build\web
```

Then open `http://127.0.0.1:5517`.

## Assets & tooling

All art is generated by dependency-free Python scripts (no Pillow/external
libraries) so it can be regenerated and tweaked without an art pipeline:

```powershell
python tools/generate_icon.py      # launcher/splash source art -> assets/branding/
python tools/generate_sprites.py   # pixel-art sprite pack -> assets/images/

dart run flutter_launcher_icons    # rebuild platform launcher icons
dart run flutter_native_splash:create  # rebuild the native splash screen
```

- Default rendering is the procedural vector "blob" style (always available,
  the fallback). An **experimental pixel-art sprite pack** lives in
  `assets/images/` and is selectable via **Settings > Experimental > Render
  Style** (Classic / Sprites). Missing sprites fall back to procedural per
  component, so a partial or replacement pack drops in safely.
- Accessibility settings: **Colorblind Assist** (per-deficiency category color
  remap + category shape glyphs on mobs and bullets), **Larger Text**, plus
  **Controls** (touch-control mode, mobile weapon-swap style, desktop mouse
  aim).

## Publish-Ready Polish Roadmap

### First Impression

- Keep the animated saliva-biomarker loading screen short, responsive, and
  readable on mobile.
- Add a title-screen transition from loading into the mission hub.
- Replace placeholder launcher icons and splash art with final branded assets.
- Tune the animated round-intro briefing for each biome and boss round.

### Gameplay Feel

- Add clearer hit flashes, enemy death bursts, and boss attack telegraphs.
- Add small reward animations after quizzes, weapon unlocks, and gold-shop
  purchases.
- Make boss phases more distinct with named attack patterns and audio cues.
- Tune round pacing with early breathing room, mid-round spikes, and final
  cleanup pressure.

### Education

- Expand the enemy dictionary with PDAC terms, saliva biomarkers, and simple
  research notes.
- Add quiz feedback that explains why each answer is correct.
- Expand the boss recap cards that connect boss fights to the
  saliva-detection storyline.

### Performance

- Keep audio pooled and throttled so web playback cannot stall combat.
- Keep particles, bullets, coins, and hazard clouds under fixed budgets.
- Use the F3 performance overlay when tuning high-mob rounds.
- Test long sessions in production web builds, not only debug builds.

### Release Quality

- Keep tuning the pause/settings flow for mouse, keyboard, and touch comfort.
- Keep the home-screen continue/new-run flow clear when a checkpoint exists.
- Add accessibility checks for text scale, contrast, reduce motion, and color
  identification.
- Add save-data migration tests before changing persistent data shapes.
- Create a short playtest checklist for performance, difficulty, science
  clarity, and first-time-player confusion.
